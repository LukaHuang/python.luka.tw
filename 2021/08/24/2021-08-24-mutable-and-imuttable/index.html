[Python 教學] Mutable 與 Immutable 是什麼？

今天來探討 Python 中的 Mutable 與 Immutable 的概念。Mutable 顧名思意就是「可變的」，而「Immutable」顧名思義是不可變得。

Immutable 與 Mutable 是什麼意思？

在 Python 中什麼是「可變的」？什麼是「不可變的」呢？

Python 中 Mutable 定義，物件的「值」可以被改變。反之，Immutable 就是物件的值不可以被改變。

在 Python 中的一切都是物件

延續剛剛的話題，雖然我們已經知道可變物件與不可變物件。

Everything Is an Object by DiveIntoPython

這是什麼意思呢？ 在 Python 中你所使用的一切東西都是物件。我們來試試看一下 Python 中的資料型別。

說明：

  1. 使用 id(object) 可以找到物件 id。
  2. 使用 __class__ 特殊方法可以找到物件是由哪個類別(class)生成的。

由此可知，在 Python 中即使是基本型別也是物件。包括函式的一切都是物件。

Immutable Object 不可變物件

直譯「不可被改變的物件」，是什麼意思呢？

當物件生成後，值(Value)無法被改變。

例如：先將 100 指派給 a ，接著我想把 a 的值改成 101 ，我們來看看會發生什麼事情。

當 a 是 100 的時候，物件 id 是 4487734736。當我想重新指派 101 給 a 的時候，因為整數資料型別(Integer)，他是一種 Immutable Object，值無法被改變。
所以，當我們指派給 101 給 a 後，a 的 id 由 4487734736 變成了 4487734768，產生了一個新的物件。
除了整數 int 以外，在 Python 中以下的型別也是 immutable 的

Mutable Object 可變物件

接下來來談談「可變的物件」，那麼什麼東西是可變的呢？同理，此物件的值可以被改變。

我們用字典 (dict) 資料型別來演示，當我們創造出一個字典資料結構的時候，裡面的內容是可以被改變的，而且外部的 id 卻不會改變。這樣的特性就跟 JS 之類的動態語言比較相似。

小結

複習一下，這篇學到的是 Immutable Object 不可變物件，物件的值不可被改變。當被改變時有兩種情況，一是會發生錯誤或，二是產生一個新的物件。

mutable Object 可變物件，物件的值可被改變，可重新賦值。id 在大多數的情況下不會被改變。那麼什麼時候會被改變呢？
